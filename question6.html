difference between HTML & HTML 5

HTML HTML 5
  • In older versions of HTML, you often saw a Document Type Definition
    (DOCTYPE) declaration that looked like
    <!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN">.
    		•
  • HTML5 has a simplified DOCTYPE declaration: <!DOCTYPE html>.
    		•
  • Traditional HTML had a limited set of structural elements like <div>, <span>, <table>, etc.
    		•
  • HTML5 introduced new semantic elements, such as <header>, <footer>, <nav>, <article>,
    <section>, and <aside>. These elements provide a more meaningful way to structure content and
    improve the document's readability for both developers and browsers.
    		•
  • Older versions of HTML required third-party plugins (like Flash) to embed audio and video.
    		•
  • HTML5 includes native support for <audio> and <video> elements, making it easier
    to embed multimedia content without relying on external plugins. This contributes
    to better cross-browser compatibility and improved performance.
    		•
  • Traditional HTML forms had limited input types and attributes.
    		•
  • HTML5 introduced new input types like date, email, tel, and url. It also brought new attributes like placeholder,
    required, and pattern, providing more options for form validation and improving the user experience.
    		•
  • Older HTML versions lacked built-in APIs for features like offline web applications, geolocation, and drag-and-drop functionality.
    		•
  • HTML5 includes several new APIs, such as the Geolocation API, Web Storage API (localStorage and sessionStorage),
    Application Cache, and the Canvas API for drawing graphics dynamically. These APIs enhance the capabilities of web applications, allowing for a more interactive and dynamic user experience.
    		•
